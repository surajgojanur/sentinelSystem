# Roadmap Notes

## Strongest Leverage Points

### 1. Explainable Governance Logs

The app already stores structured evidence about why a response was allowed, filtered, or blocked. That is a foundation for investigation workflows, approval systems, and accountable AI operations.

### 2. Assignment + KPI + Capacity Risk

The repo already has a real operational workflow, not just chat. This creates a path toward decision support, manager intelligence, and escalation quality improvements.

### 3. Attendance Signal In The Same System

Attendance records already feed workload risk. That is rare leverage for building low-cost operational resilience features without adding large infrastructure.

### 4. Unanswered Queue + Feedback + History

The question-bank layer already captures policy/content gaps. This can become a governance improvement engine instead of a static FAQ manager.

### 5. Suspicious User Counters

The repo already tracks risky behavior at the user level. It is not yet advanced, but it is enough to grow into accountable insider-risk or misuse-review workflows.

### 6. Escalation Object Already Exists

Even though it is simple today, the escalation model is real and tied to work items. That is a strong base for human-in-the-loop risk review.

## Promising Product Directions

### Direction A

AI governance and accountability platform for internal enterprise assistants.

### Direction B

Security-aware workforce operations platform that links work performance, attendance signals, and escalation control.

### Direction C

Manager decision-support system that explains where intervention is needed and why.

## Low-Risk Next Steps

- strengthen admin-action auditability
- enrich escalation records before adding new large modules
- create cross-signal risk views combining assignments, attendance, and suspicious AI usage
- turn unanswered-query and feedback data into policy-gap reporting
- make the current core more coherent before expanding scope

## Things Not To Build Yet

- full SIEM-scale event pipeline
- heavyweight external approval engines
- complex autonomous AI agents acting on work items
- deep biometric-security claims
- generic collaboration-suite features that dilute the product

## Recommended Product Bias

Bias toward:

- explainability
- accountability
- governed decisions
- human-reviewed escalations
- cross-functional operational intelligence

Avoid bias toward:

- generic productivity SaaS
- cosmetic feature expansion
- security theater without operational proof
